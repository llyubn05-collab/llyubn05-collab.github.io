# 한 학기 동안 만든 것

이번 학기에는 GitHub Pages와 Jekyll을 이용해 개인 포트폴리오 사이트를 만들고, JavaScript와 Chart.js를 활용해 문학 텍스트를 분석하는 도구를 구현했다. 최종 사이트에는 자기소개와 프로젝트 카드가 있는 홈 화면, 셜록 홈즈 작품 비교 페이지, 고딕 소설 비교 페이지, 그리고 사용자가 직접 영어 텍스트를 붙여 넣어 단어 빈도를 확인할 수 있는 분석기 페이지를 넣었다. 셜록 홈즈 비교 페이지와 고딕 소설 비교 페이지는 미리 준비된 텍스트 파일을 불러와 두 작품의 단어 빈도를 비교하고, 사용자 입력 분석기는 원하는 영어 텍스트를 직접 넣어 상위 20개 단어를 확인할 수 있도록 만들었다.

## 겪었던 문제 및 해결

구현 과정에서 가장 많이 겪은 문제는 함수 이름의 사소한 오타에서 비롯된 오류였다. 처음에는 화면만 보고 왜 결과가 나오지 않는지 알기 어려웠지만, 개발자 도구 Console 탭을 확인하면서 오류 메시지를 읽는 방식으로 문제를 찾았고 디버깅 하는 방법을 배웠다. 또한 제목을 쓸 때 YAML 프론트매터에서 title 값에 특수문자를 따옴표 없이 쓸 경우 잘못 해석되어 문제가 생기는 경우가 많아 따옴표를 잘 써주었더니 경로 문제가 해결되었다. 

## 분석에서 발견한 것

 흥미로웠던 점은 단어 빈도만 보아도 작품의 분위기와 관심사가 어느 정도 드러난다는 것이었다. 예를 들어 같은 셜록 홈즈 시리즈더라도 상위 단어에서 어떤 작품은 말하기와 진술에 관련된 단어, 어떤 작품은 추리 전개 방식과 관련한 단어가 상위에 나타나는 등 차이를 보였다. 고딕 소설 비교에서는 두 작품이 모두 어둡고 불안한 분위기를 공유하지만, 같은 고딕 소설이라고 해서 완전히 같은 방식으로 공포를 만드는 것은 아니라는 점이 흥미로웠다.
 다만 단어 빈도 분석만으로 작품의 의미를 모두 설명할 수는 없다는 한계도 느꼈다. 많이 등장한 단어가 반드시 가장 중요한 주제를 뜻하는 것은 아니고, 반대로 적게 등장한 표현이 작품 전체의 핵심 장면이나 상징과 연결될 수도 있다. 따라서 단어 빈도 분석은 문학 작품의 해석을 완성하는 방법이라기보다는, 작품을 다시 읽기 위한 출발점에 가깝다고 생각한다.
